# カードゲームAR配信アプリ 開発ロードマップ

## プロジェクト概要

**目的**: デュエルマスターズの観戦体験を向上させる  
**コンセプト**: スマホカメラでカードを認識し、AR空間にキャラクターを投影。カードのタップ・アンタップ・ゾーン移動に連動したアニメーションを付け、YouTube配信に対応する

### 技術スタック

| レイヤー | 技術 |
|---|---|
| カード認識 | YOLOv8 OBBモード（回転バウンディングボックス） |
| バックエンド | Python + FastAPI |
| フロントエンド | Flutter |
| AR | ar_flutter_plugin（ARCore / ARKit） |
| アノテーション | Roboflow |
| 3Dモデル（余裕があれば） | glTF形式 |

---

## Phase 1 — データ収集 & カード認識

**目安期間**: 3〜4週間

### やること

1. **スクレイピング**
   - デュエルマスターズ公式サイトからカード画像を収集
   - 参考: https://note.com/wabisuke94/n/n6cd99789e7aa
   - 最初は有名カード10〜20枚に絞る（ボルメテウス、ボルバルザーク等）
   - パイプライン全体を通すことを優先し、全カード対応は後回し

2. **アノテーション**
   - Roboflowでバウンディングボックスをラベル付け
   - YOLOv8用データセットとしてエクスポート
   - データ拡張（albumentations）で水増しして精度向上

3. **YOLOv8 OBBで学習**
   - OBBモード（回転バウンディングボックス）を使用
   - 回転角を出力させることでタップ・アンタップ判定に使う
   - 90°回転 → タップ状態として判定

### 注意点

- 光の反射・角度ブレに弱いので、様々な角度・照明条件の画像をデータに含める
- まず手動で画像20枚集めてもOK。スクレイピングは後からでも間に合う

## Phase 2 — FlutterからAPI連携

**目安期間**: 1〜2週間

### やること

1. **FastAPIでYOLOv8をAPI化**
   ```
   POST /detect
   body: { image: base64 }
   response: { cards: [{ id, bbox, angle, is_tapped }] }
   ```

2. **Flutter側でカメラ映像を送信**
   - camera パッケージでフレームをキャプチャ
   - 一定間隔（例: 500ms）でAPIにPOST

3. **認識結果をUIに表示して確認**
   - カード上に認識枠を重ねて表示
   - タップ状態の判定が正しいか目視確認

### アーキテクチャ

```
[Flutterアプリ]
      |
      | カメラ映像（base64）
      v
[Python FastAPI]
      |
      | YOLOv8推論
      v
[カードID・座標・角度・タップ状態]
      |
      v
[Flutterアプリ] → AR描画へ
```

## Phase 3 — AR投影（2Dキャラ）

**目安期間**: 2〜3週間

### やること

1. **ar_flutter_pluginで平面アンカーを設置**
   - カードの座標をもとに平面アンカーを張る
   - まずAndroidのみで動かし、iOS対応は後から

2. **ビルボードとしてキャラを立てる**
   - 最初は3Dモデルではなく、2Dイラストを板ポリ（ビルボード）として表示
   - カードの画像からキャラ部分を切り出してPNG透過画像を用意

3. **ゾーン移動への追従**
   - カードのbboxが変化したらアンカー位置を更新
   - フレーム間の座標変化をスムージングしてカクつきを防ぐ

### 注意点

- `ar_flutter_plugin`が古いFlutterバージョンで動かない場合は、FlutterUnityWidget（Unity統合）も検討
- まずAndroidのみで動作確認してからiOS対応

---

## Phase 4 — タップ検出 & アニメーション

**目安期間**: 2〜3週間

### やること

1. **OBBの回転角でタップ判定**
   - YOLOv8 OBBの出力角度を取得
   - 閾値（例: ±45°）でタップ / アンタップを分類

2. **状態管理**
   - 各カードのタップ状態をFlutter側で保持
   - 状態変化（アンタップ→タップ、タップ→アンタップ）を検出

3. **アニメーションを連動**
   - タップ時: キャラがお辞儀するアニメーション
   - アンタップ時: 起き上がるアニメーション
   - ゾーン移動時: 移動モーション

---

## Phase 5 — YouTube配信対応（余裕があれば3D）

**目安期間**: 別途検討

### やること

1. **配信方法の検討**
   - スマホの画面をそのまま配信（OBS仮想カメラ or スマホ直配信）
   - ARオーバーレイが乗った状態で配信に流れることを確認

2. **配信用オーバーレイUI**
   - カード名・コスト・HPなどの情報を画面上に表示
   - 観戦者向けのゲーム状況テロップ

3. **3Dモデル挑戦（余裕があれば）**
   - glTF形式のモデルをARに乗せる
   - 自作が難しい場合はVRoid等で簡易モデルを作成

## リスクと対策

| リスク | 対策 |
|---|---|
| スクレイピングがサイト構造で詰まる | 最初は手動で20枚収集してパイプラインを優先 |
| 光反射・角度でYOLOv8の精度が出ない | データ拡張で多様な条件を学習させる |
| ar_flutter_pluginが環境で動かない | まずAndroidのみ、問題あればFlutterUnityWidgetも検討 |
| カードが重なって認識できない | 検出信頼度スコアで低いものを除外するロジックを入れる |

---

## 学習できる技術

- **YOLOv8**: 物体検出・回転バウンディングボックス（OBB）
- **FastAPI**: PythonでのAPI構築
- **ar_flutter_plugin**: ARCore / ARKit のFlutter統合
- **Roboflow**: アノテーション・データセット管理
- **glTF / 3Dモデル**（Phase 5）
