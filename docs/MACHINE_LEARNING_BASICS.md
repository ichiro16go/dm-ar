## 機械学習とは

### シンプルな説明

**機械学習 = コンピュータが「パターン」を自動で学ぶ**

例：
```
❌ 従来のプログラミング
画像を分析する → 手動でロジックを書く
if color == "red" and size > 100:
    print("赤いカード")

✅ 機械学習
大量のカード画像を見せる → モデルが「カード」のパターンを勝手に学ぶ
モデル.推論(新しい画像) → "このは多分カード"
```

### 3つのステップ

```
[データ収集] → [訓練] → [推論]
     ↓          ↓        ↓
画像1000枚   モデルを  新しい画像を
集める      学習させる 認識させる
```

### 用語

| 用語 | 意味 |
|-----|------|
| **データセット** | 訓練に使用する画像・ラベルのセット |
| **モデル** | 学習済みの「脳」。重み付けパラメータの集合 |
| **訓練（Training）** | モデルに大量のデータを見せて学習させる |
| **推論（Inference）** | 訓練済みモデルに新しいデータを入力して予測 |
| **損失（Loss）** | 予測の間違い度合い（少ないほど良い） |
| **精度（Accuracy）** | 正解率。90%なら100個中90個正解 |

## 転移学習（ファインチューニング）

### なぜ転移学習が必要？

```
❌ ゼロからの学習（フルスクラッチ）
- 必要なデータ: 100万枚+
- 時間: 数ヶ月
- GPU: 複数枚必須
- 費用: 数百万円

✅ 転移学習
- 必要なデータ: 100～1000枚
- 時間: 1時間～1日
- GPU: 1枚でOK
- 費用: ほぼ無料（個人マシン程度）
```

### 転移学習の仕組み

```
[事前訓練モデル]
ImageNet で訓練済み
- 「犬」「猫」「自動車」など
  1000万枚で学んだ
                       ↓
                   重みを再利用
                       ↓
          [ファインチューニング]
          カード検出用に最後の層を
          カスタマイズして再訓練
          （100～200枚程度でOK）
                       ↓
                   [最終モデル]
                  カード検出用に
                  最適化された
                   YOLOv8
```

### 何が起きているか？

```
事前訓練モデルが学んだもの:
- Level 1: 「エッジ」「コーナー」など基本的な形状
  （全ての物体認識で共通）
- Level 2: 「四角形」「のような図形」など中レベルパターン
- Level 3: 「顔っぽい形」など高レベルパターン

ファインチューニング時:
- Level 1, 2 はそのまま使用（学習時間短縮）
- Level 3 と最終分類層だけ「カード検出」に特化させてリトレーニング
```

## YOLOv8について

### YOLOvって何？

**YOLO** = "You Only Look Once" の略

- **YOLO v1** (2015) - 物体検出の革命
- **YOLO v2** (2016) - より速く、より正確に
- ...
- **YOLO v8** (2023) - 最新版、実用的

### YOLOv8でできること

```
1. 物体検出（Object Detection）
   画像内の物体の位置と種類を検出

2. OBB モード（Oriented Bounding Box）
   ← これが私たちが使う
   
   回転したバウンディングボックスで検出
   通常のBBoxが軸並行四角形に対し、
   OBBは任意の角度に対応
   
   → カードは傾いていることが多いので最適！
```

### 動作フロー

```
[入力画像]
    ↓
[YOLOv8フォワードパス]
  - 複数のスケール（大きさ）で検出
  - 高速に処理（リアルタイム可能）
    ↓
[出力]
- bbox: [x, y, width, height]      (位置・大きさ)
- angle: 45.3°                      (回転角)
- confidence: 0.95                  (信頼度 0～1)
- class: "card"                     (分類)
```

## 私たちの実装フロー

### Phase 1: カード認識

```
1️⃣  データ収集
    ├─ デュエルマスターズカード画像 100～200枚
    ├─ 様々な角度・照明条件で撮影
    └─ Roboflow にアップロード

   💡 学習ポイント:
      - なぜ多様なデータが重要？
        → モデルは「訓練データと似た」状況に強い
        → 1種類の照明だけでは本来使えない

2️⃣  アノテーション
    ├─ Roboflow で バウンディングボックスラベル付け
    ├─ OBB 形式（回転角を記録）
    └─ データセットをエクスポート → YOLOv8形式に変換

   💡 学習ポイント:
      - なぜラベル付けが必須？
        → 教師あり学習では「正解」が必要
        → 「どの部分がカード」か教える

3️⃣  訓練
    ├─ YOLOv8 OBB モデルをロード（事前訓練済み）
    ├─ あなたのカード画像で ファインチューニング
    │  └─ 最後の層だけを「カード」に特化させる
    └─ 100 エポック（同じデータを100周）訓練

   💡 学習ポイント:
      - エポック（epoch）とは？
        → 全訓練データを1周通すことを「1エポック」
        → 複数周することで学習を深める
        → でも周りすぎると「過学習」（過度にオーバーフィット）

4️⃣  推論テスト
    ├─ 新しいカード画像を与える
    ├─ モデルが「カード」と「回転角」を検出
    └─ 結果を画像に描画して確認

   💡 学習ポイント:
      - 訓練データと異なる画像でテスト重要
        → 訓練データで完璧でも、新しいデータには弱いことが多い
```

## 一般的な落とし穴

### 1. データが少なすぎる
```
❌ 画像 10 枚だけ
   → 過学習（その10枚には完璧だが、他は認識できない）

✅ 画像 100～500 枚 + 多様性
   → ファインチューニングには十分
```

### 2. データの偏り
```
❌ 全て同じ照明・角度
   → 訓練と異なる条件に弱い

✅ 顔上・光源多様・回転角度多様
   → 実世界で堅牢
```

### 3. エポック数の誤り
```
❌ 1000 エポック（オーバーフィット）
   ✓ 訓練データ: 99% 精度
   ✗ テストデータ: 60% 精度

✅ 100 エポック（Early Stopping で自動停止）
   ✓ 訓練データ: 95% 精度
   ✓ テストデータ: 90% 精度（バランス型）
```

## 重要な概念: 転移学習 vs フルスクラッチ

| 項目 | フルスクラッチ | 転移学習 |
|------|-------|-------|
| 初期状態 | ランダムな重み | ImageNet事前訓練済み |
| 必要データ | 100万★ | 100～1000 |
| 訓練時間 | 数ヶ月 | 数時間～1日 |
| GPU | 複数必須 | 1個で十分 |
| 用途 | 完全に新しいタスク | 既知の物体検出タスク |

**私たちの場合は後者（転移学習）です！**

## 次の学習テーマ

Phase 1 が終わったら：
1. **精度評価** - モデルがどれくらい正確か測定
2. **混同行列（Confusion Matrix）** - どんな間違いをしているか分析
3. **データオーグメンテーション** - 限られたデータを増やすテクニック
4. **ハイパーパラメータチューニング** - 学習率・バッチサイズの最適化
